写在最前

具有电信、移动两条宽带，通过配置openwrt策略路由，实现国内网站走电信出口，国外网站走移动出口

由于今年上半年上海电信的海外线路质量持续下降，尤其到了6月份出现了“肉眼可见”的访问障碍，严重影响了网络使用体验。（具体可以“上海电信 国外&海外”等关键字搜索到许多用户的反馈）

综合魔都网友的使用反馈，本人在 7.11 新开通了一条上海移动的宽带。现有宽带接入为

  1. 电信光纤接入 下行50M 上行10M
  2. 移动有线通同轴 下行10M 上行5M

通过试用后发现移动出国线路确实优秀，不区分闲时忙时连接国外服务器基本都是跑满带宽，但访问国内网站总有不顺畅的感觉——以日常应用来说，诸如 微博图片、网易音乐等使用CDN的网站都出现一定程度的拖慢。

恰笔者目前主力路由器为 网件 wndr3700 v4 （之前一直安装ddwrt系统当作普通家用无线路由使用），有相应的较为稳定的openwrt系统，因此笔者决定折腾一下家庭的网络路由配置，优化国内外网络质量。

如果你也有相类似的情况和需求，但相关的网络技术知识一般，你可以参考本文的思路和方法，实现开头说到的效果。

同时本文也是笔者摸索学习，不断撞墙，不断重来的过程，记录备份免得以后再绕路。

根据ip所在地作策略路由

1）使用iptables作标识
使用模块

iptables-mod-conntrack-extra iptables-mod-geoip iptables-mod-nat-extra

  • geoip模块在opkg默认仓库有 注意安装后要自己添加ip地址库文件 具体请google

具体语法 

iptables -t mangle -I PREROUTING -m geoip -p tcp ! --dst-cc CN -j MARK --set-mark 1
  • -t mangle 标记用的表
  • -I PREROUTING 防火墙最外层（最先处理）的链
  • -m geoip 调用geoip模块
  • ! --dst-cc CN 表示目的地为中国的取反
  • -j MARK --set-mark 1 调用标记插件来给滤出的ip作标识
  • 如果只是想区分tcp包可以加上 -p tcp 参数

2）标识ip 归入区分路由表
具体语法 

ip rule add fwmark 1 priority 1990 table yd
  • “将具有标识〈1〉的所有 ip 交给〈yd〉这张路由表处理”
  • priority 1990 表示优先级 越小越高

3）配置路由表走特定出口
具体语法 

ip route add default via 192.168.100.1 dev eth0.2 table yd
ip route add 192.168.6.0/24 dev br-lan table yd
  • 〈yd〉路由表的出口网关是192.168.100.1（移动WAN接口）
  • 路由来自LAN子网的访问 交给LAN的网关
  • 之前就是少了第二句所以总是出不去，基本原理是因为来自LAN内网的访问是不能越级去访问WAN的网关的，得一层层出去
